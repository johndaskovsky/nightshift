## MODIFIED Requirements

### Requirement: Command files are installer-generated
The system SHALL treat the Nightshift Claude Code skill files as installer-generated artifacts rather than manually-committed source files. Each skill SHALL be written by `nightshift init` from templates bundled in the CLI package.

#### Scenario: Skills are available after init
- **WHEN** a user runs `nightshift init` in a fresh project and then opens Claude Code
- **THEN** all six Nightshift skills SHALL be available: `/nightshift-create`, `/nightshift-start`, `/nightshift-archive`, `/nightshift-add-task`, `/nightshift-test-task`, `/nightshift-update-table`

#### Scenario: Skills match current behavior after generation
- **WHEN** skill files are generated by the installer
- **THEN** the generated skill files SHALL produce the behavior described by the requirements in this spec

### Requirement: Claude Code skill surface for all six commands
The system SHALL provide a Claude Code Skill for each of the six Nightshift commands. Each skill SHALL produce the behavior described by its dedicated requirement in this spec (shift directory layout, `manager.md` and `table.csv` mutations, archive behavior).

#### Scenario: Skills available after install
- **WHEN** a user runs `nightshift init` in a fresh project and then opens Claude Code
- **THEN** all six Nightshift skills SHALL be available: `/nightshift-create`, `/nightshift-start`, `/nightshift-archive`, `/nightshift-add-task`, `/nightshift-test-task`, `/nightshift-update-table`

### Requirement: Skills accept shift name argument
The system SHALL allow each skill to accept a shift name as an argument via Claude Code's `$ARGUMENTS` (or `$0`) substitution.

#### Scenario: Skill receives shift name
- **WHEN** a user invokes `/nightshift-start my-batch-job`
- **THEN** the skill body SHALL receive `my-batch-job` substituted for `$ARGUMENTS` (or `$0`) at render time

#### Scenario: Skill prompts when argument omitted
- **WHEN** a user invokes a skill that requires a shift name (e.g., `/nightshift-start`) without providing one
- **THEN** the skill SHALL list the available shift directories and ask the user to pick one (using `AskUserQuestion` when more than one is available, auto-selecting when only one is available)
